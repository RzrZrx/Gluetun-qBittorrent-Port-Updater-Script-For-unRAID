## Contributing

When responding to issues:
1. Create a comprehensive response document here
2. Use it as a template for the GitHub issue reply
3. Update this README with a link to the new document
4. Consider adding the solution to the main troubleshooting section if it's common

## Format

Each issue document should include:
- **Problem Description** - What error/symptom the user sees
- **Common Causes** - List of typical root causes
- **Diagnostic Steps** - How to identify the specific cause
- **Solutions** - Step-by-step fixes
- **Prevention** - How to avoid the issue in the future
- **Related Issues** - Links to similar problems
